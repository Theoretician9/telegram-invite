## 2025-05-21: Переход на асинхронную публикацию постов в Telegram (python-telegram-bot 22.x+)

- Пользователь столкнулся с ошибкой 'coro' object has no attribute ...' при публикации постов через Celery-задачи.
- Ассистент диагностировал, что причина — использование python-telegram-bot версии 22.x+, где все методы стали асинхронными.
- Был предложен выбор: откатиться на 13.15 (синхронная), либо переписать код под async-стиль.
- Принято решение переписать код под асинхронный вариант.

### Изменения в tasks.py:
- Функция publish_post_task теперь использует async-функцию send() и вызывается через asyncio.run(). Вся отправка сообщений — только через await bot.send_message(...).
- Добавлено подробное логирование ошибок публикации (chat_id, токен, длина текста, тип ошибки), а также обработка типовых ошибок (чат не найден, бот заблокирован, недостаточно прав).
- Логика force-публикации теперь работает корректно: можно повторно публиковать посты, даже если они уже были опубликованы.
- Функция autopost_task также переписана на асинхронный стиль: все отправки сообщений через await bot.send_message(...) внутри async-функции send_all(), вызов через asyncio.run(send_all()).
- Добавлена задержка между постами через await asyncio.sleep(5).

### Диагностика и рекомендации:
- Проведён аудит кода: все места, где используется Bot из python-telegram-bot, переведены на асинхронный стиль.
- В других модулях (parser.py, alerts.py) публикация сообщений либо не используется, либо реализована через requests (alerts.py), что не требует изменений.
- Для старых проектов возможен откат до python-telegram-bot 13.15, если требуется синхронная работа.
- Для новых проектов рекомендуется использовать только async-стиль и актуальные версии библиотеки.

### Инструкция по обновлению:
- После внесения изменений требуется перезапустить Celery-воркеры, чтобы они увидели новый код.
- Для публикации постов и автопостинга теперь используется только асинхронный стиль.
- В случае появления новых ошибок публикации — смотреть логи, где теперь выводится подробная диагностика.

## 2025-05-22: Ежедневный автопостинг по расписанию, поддержка остановки задачи

- Автопостинг теперь работает как бесконечный цикл: каждый день в указанные часы (по Москве) генерируется и публикуется следующий пост по выжимке.
- Используется московское время (`Europe/Moscow`) для всех расчётов расписания.
- При повторном запуске автопостинга предыдущая задача получает сигнал остановки (через Redis), завершает работу, и стартует новая задача с новым расписанием.
- В Redis хранится статус автопостинга (`autopost_status`) и task_id активной задачи (`autopost_task_id`).
- Весь процесс автопостинга подробно логируется: старт задачи, расписание, каждый слот публикации, ожидание, публикация поста, завершение задачи.
- Теперь всегда работает только одна активная задача автопостинга на книгу, и расписание можно менять "на лету" без перезапуска Celery.
